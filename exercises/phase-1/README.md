# Phase 1 실습 과제 — HTML & CSS

Phase 1 문서 학습과 병행하는 2단계 실습 과제다. 각 단계는 [학습 기획](../../plan/phase1.md)의 주차 배분과 연동되며, 완성 기준(Definition of Done)을 체크리스트로 명시한다.

## 과제 A — 자기소개 페이지 (1~2주차)

문서 진도에 맞춰 같은 페이지를 4단계로 발전시킨다. **단계마다 커밋을 남겨** 발전 과정을 이력으로 확인할 수 있게 한다.

### A-1. 시맨틱 마크업 (1-2 학습 후)

CSS 없이 HTML만으로 자기소개 페이지를 작성한다. 소개, 경력/기술 스택, 연락처(폼 포함) 섹션을 갖춘다.

- [ ] 랜드마크 요소(header/main/footer 등)로 골격을 잡았다
- [ ] h1은 1개, heading 레벨을 건너뛰지 않았다
- [ ] 연락처 폼의 모든 입력에 label과 name이 연결되어 있다
- [ ] 브라우저 리더 모드에서 본문이 올바르게 추출된다
- [ ] [Nu HTML Checker](https://validator.w3.org/nu/) 오류 0건

### A-2. 기본 스타일링 (1-3 학습 후)

- [ ] `box-sizing: border-box` 전역 리셋을 적용했다
- [ ] 텍스트 크기는 rem 기반이다 (px 고정 없음)
- [ ] 선택자는 클래스 중심이고 `!important`가 없다
- [ ] body의 상속을 활용해 폰트/색을 한 곳에서 선언했다

### A-3. 레이아웃 (1-4 학습 후)

- [ ] 페이지 골격(헤더/본문/사이드 영역/푸터)을 Grid로 구성했다
- [ ] 구획 내부 정렬(내비게이션, 기술 스택 태그 등)은 Flexbox를 사용했다
- [ ] 간격은 개별 마진 대신 `gap`으로 관리했다
- [ ] absolute는 겹쳐 얹는 장식(배지 등)에만 사용했다

### A-4. 반응형 (1-5 학습 후)

- [ ] viewport 메타 태그가 있다
- [ ] 모바일 퍼스트(`min-width` 쿼리)로 작성했다
- [ ] 브레이크포인트 2개 이상, 320px~1440px 연속 구간에서 깨지지 않는다
- [ ] 이미지에 width/height(또는 aspect-ratio)가 지정되어 있다

## 과제 B — 랜딩 페이지 클론 (3주차, 1-6·1-7 학습 후)

실제 서비스(카페, 쇼핑몰, SaaS 등)의 랜딩 페이지를 하나 골라 반응형으로 클론하고 GitHub Pages로 배포한다. 픽셀 단위 복제가 아니라 **구조 분석 → 재구현**이 목적이다: 클론 대상을 먼저 종이에 랜드마크/Grid 영역으로 분해해 보고 시작한다.

### 완성 기준 (Definition of Done)

**구조·스타일**

- [ ] 과제 A의 체크리스트(시맨틱, border-box, rem, Grid+Flexbox, 모바일 퍼스트)를 모두 충족한다
- [ ] 색·간격·글꼴 크기를 커스텀 프로퍼티(디자인 토큰)로 선언하고 컴포넌트는 토큰만 참조한다
- [ ] hover/focus 인터랙션이 있으며, 포커스 표시는 `:focus-visible`로 처리했다
- [ ] transform/opacity 기반 트랜지션이 1개 이상 있고, `prefers-reduced-motion`을 존중한다

**접근성·품질**

- [ ] 마우스 없이 키보드만으로 모든 기능(내비게이션, 폼, 버튼)을 완주할 수 있다
- [ ] 아이콘 전용 버튼에 접근 가능한 이름이 있고, 장식 이미지는 트리에서 제외했다
- [ ] Lighthouse Accessibility **90점 이상**, 콘솔 오류 0건
- [ ] 텍스트 대비가 WCAG AA(4.5:1)를 충족한다 (DevTools 색상 피커로 확인)

**배포**

- [ ] GitHub Pages로 배포되어 URL로 접근 가능하다
- [ ] README에 클론 대상, 구조 분석(랜드마크/Grid 스케치), 구현하며 배운 점을 기록했다

## 진행 팁

- 막히면 해당 문서의 "확인 문제"를 다시 풀어 본다 — 과제에서 만나는 문제의 대부분(마진 겹침, z-index, 명시도)은 확인 문제와 같은 유형이다.
- 디버깅은 추측 대신 DevTools로: 레이아웃은 Elements의 박스 시각화, 스타일 충돌은 Styles 패널의 취소선, 접근성은 Accessibility 패널.
