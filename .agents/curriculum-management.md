# 커리큘럼·기획·실습 관리 지침

이 문서는 `ROADMAP.md`, `PROGRESS.md`, `plan/`, `exercises/`를 변경하거나 챕터의 파일·범위를 조정할 때 적용한다. 교육 콘텐츠 본문도 수정하면 [콘텐츠 집필 지침](content-writing.md)을 함께 따른다.

## 자료별 책임

- `ROADMAP.md`는 전체 목표, Phase·부록의 순서와 범위, 문서 파일 경로의 기준 문서다.
- `PROGRESS.md`는 문서 작성 진행 상태와 미완료 TODO의 기준 문서다. Phase·부록 구성은 ROADMAP과 일치시킨다.
- `plan/`은 Phase와 부록의 상세 기획, 문서 간 의존 관계, 조사·작업 노트를 담는다. ROADMAP과 충돌하면 ROADMAP을 우선한다.
- `exercises/`는 각 Phase의 학습 결과를 실제 구현·분석·측정으로 검증하는 요구사항과 완료 기준을 담는다.
- `docs/`는 배포되는 교육 콘텐츠 원본이다. 경로와 주제는 ROADMAP의 해당 항목에 맞춘다.

## 변경 절차

1. 관련 Phase 또는 부록의 ROADMAP 항목과 `plan/` 문서를 먼저 확인한다.
2. 앞뒤 챕터와 관련 실습을 확인해 선수 지식, 후속 심화, 중복 설명의 경계를 정한다.
3. 문서 추가·삭제·이름 변경·범위 변경이 필요하면 ROADMAP을 먼저 정리하고 관련 기획·실습·링크를 맞춘다. 새 Phase·부록 디렉터리가 생기면 [저장소 운영 지침](repository-operations.md)에 따라 `docs/.vitepress/navigation.ts`의 발견 패턴·레이블·정렬도 작업 범위에 포함한다.
4. 단순 오탈자, 표현 개선, 범위 안의 정확성 보완에는 PROGRESS 진행 상태 갱신을 요구하지 않는다.
5. 학습 목표나 산출물의 의미가 달라질 때만 관련 실습 요구사항과 완료 기준을 함께 수정한다.

## 문서 간 연결

- 앞선 챕터에서 다룬 내용은 반복하지 말고 상대 링크로 연결한다.
- 다른 챕터나 Phase에서 심화할 주제는 현재 문서의 범위와 위임 지점을 명시한다.
- 새 파일은 해당 디렉터리의 숫자 접두사와 명명 패턴을 따른다.
- 문서 수나 완료 상태가 실제로 변할 때만 PROGRESS의 진행 현황을 갱신한다.

## 진행 현황 TODO 관리

- `PROGRESS.md`의 진행 현황에는 아직 완료되지 않은 작업만 체크되지 않은 TODO 항목(`- [ ]`)으로 남긴다.
- TODO에는 완료에 필요한 작업 범위를 구체적으로 적고, 진행 현황 표의 미완료 상태와 일치시킨다.
- 작업이 완료되면 진행 현황 표의 상태를 갱신하고 해당 TODO 항목은 체크된 상태로 보존하지 말고 삭제한다.
- 완료된 작업, 기한이 정해지지 않은 상시 유지보수, 반복 점검 사항은 TODO 목록에 남기지 않는다.

## 문서 유형별 기준

- Phase·부록 챕터는 [콘텐츠 집필 지침](content-writing.md)의 기본 구조를 따른다.
- 기획 문서는 의사결정과 의존 관계가 드러나면 되며 챕터 구조를 따를 필요가 없다.
- 실습 문서는 학습 목표, 수행 요구사항, 관찰·측정 방법, 완료 기준이 서로 연결되어야 한다.
- VitePress 내비게이션이나 빌드 동작까지 영향을 주면 [저장소 운영 지침](repository-operations.md)을 함께 따른다.

## 완료 전 점검

- ROADMAP의 주제, 순서, 경로, 문서 수가 실제 파일과 일치하는가?
- PROGRESS의 문서 수와 완료 상태가 ROADMAP 및 실제 파일과 일치하는가?
- 새 Phase·부록과 문서가 VitePress nav/sidebar의 올바른 위치에 실제 포함되는가?
- 기획과 실습이 현재 학습 목표 및 챕터 범위를 가리키는가?
- 파일 이름을 바꿨다면 모든 상대 링크와 참조를 갱신했는가?
- 단순 콘텐츠 교정에 불필요한 상태·범위 변경을 섞지 않았는가?
- 진행 현황 TODO에 미완료 작업만 있으며, 완료된 항목은 삭제했는가?
