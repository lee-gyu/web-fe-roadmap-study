# 저장소·사이트 운영 지침

이 문서는 VitePress, 패키지, 도구 버전, CI·배포 설정을 변경하거나 문서 변경의 사이트 동작을 검증할 때 적용한다. 교육 콘텐츠나 커리큘럼도 바꾸면 해당 지침을 함께 따른다.

## 도구와 의존성

- 로컬 도구 버전은 `mise.toml`, 패키지 매니저·엔진 조건·스크립트·의존성은 `package.json`, CI의 실행 버전은 `.github/workflows/deploy-pages.yml`에서 확인한다.
- 패키지 작업에는 pnpm을 사용한다. 버전 숫자를 이 지침에 복제하지 않는다.
- 도구 버전을 변경하면 로컬 설정, 패키지 메타데이터, CI 설정이 서로 호환되는지 확인하고 필요한 파일을 함께 갱신한다.
- 의존성을 변경하면 lockfile을 함께 갱신하고 기존 `package.json` 스크립트로 검증한다.

## Markdown과 VitePress

- 교육 문서의 canonical 원본은 `docs/` 아래 Markdown이다. 사이트를 위해 다른 콘텐츠 디렉터리에 복제하지 않는다.
- `docs/.vitepress/navigation.ts`는 지원하는 이름 패턴의 Phase·부록 디렉터리, 숫자 접두사가 있는 파일명, 문서의 첫 번째 H1을 읽어 내비게이션과 사이드바를 만든다. 디렉터리가 정규식과 설정에 포함되지 않아도 VitePress page build 자체는 성공할 수 있으므로 자동 발견을 가정하지 않는다.
- 기존에 인식되는 Phase·부록 안의 일반 챕터 추가·수정에는 수동 사이드바 목록을 만들지 않는다. 대신 새 문서가 기존 그룹에 실제 포함되는지 확인한다.
- 새 Phase·부록 디렉터리나 `phase-5a`처럼 기존 숫자 Phase의 보조 단계를 추가하면 `navigation.ts`의 디렉터리 패턴, 레이블, 정렬 순서, 그룹 식별자를 함께 점검·수정한다. 이 점검은 새 그룹 추가 작업의 선택 사항이 아니라 완료 조건이다.
- 기존 상대 링크, 코드 블록, 표, `<details>` 기반 확인 문제의 렌더링을 보존한다.
- 사이트 설정 변경을 콘텐츠 집필 규칙 변경으로 간주하지 않는다.

## 외부 참고 자료

`hacker-laws/`는 부록 A 분류의 원문 참고 자료다. 명시적으로 요청되지 않은 일반 콘텐츠·사이트·저장소 변경에서는 수정 대상과 일괄 처리 범위에서 제외한다.

## 검증

- 작업 전후 `git status`를 확인해 요청 범위 밖의 변경을 보존한다.
- 문서나 VitePress 변경 후 `pnpm docs:build`를 실행해 파일명, 첫 번째 H1, 내부 링크, 사이트 빌드를 확인한다.
- 새 Phase·부록 또는 챕터 묶음을 추가했다면 `getPhaseGroups()` 결과와 생성된 nav/sidebar에서 새 그룹의 레이블, 문서 수, 순서, 링크를 확인한다. page와 sitemap 생성만으로 내비게이션 포함을 검증한 것으로 간주하지 않는다.
- 변경 유형에 맞는 기존 스크립트를 우선 사용하고, 실질적인 검증을 하지 않는 placeholder 스크립트를 성공 기준으로 삼지 않는다.
- `git diff --check`로 공백 오류를 확인하고, 실행하지 못한 검증은 결과에 명시한다.
